## 概述

v0.5 是 ash 从"数据感知 shell"向 **Agent 就绪 shell** 跨越的关键版本。本版完成了 RoadMap 的三个里程碑（MS1 + MS2 + MS3），让 ash 成为任何 AI/Agent 框架都能直接调用的**安全、可解析、可脚本化**的命令执行工具。

v0.4 让人用着顺手（结构化管道、表格渲染、POSIX 合规）；v0.5 让 **AI 用着更顺手**——一次性执行命令并拿到结构化 JSON、在沙盒里安全执行、用 try/catch + system() 写真实自动化脚本。

## ✨ 亮点功能

### MS1：Agent 可调用（Plan 007）

AI 能 `ash -c "ls | sort -w size" --json` 一次性执行并拿到结构化结果。

```
# 默认：渲染表格（人读）
ash -c "ls | sort -w size"

# --json：管道末端序列化成 JSON（Agent 读）
ash -c "ls | sort -w size" --json
# → [{"name":"...","size":...}, ...]

# 也支持 -s / 脚本文件，每条命令输出一行 NDJSON
ash -s --json < deploy.ash
```

- `--json` 全局 flag（可在命令行任意位置）
- 文本输出 → JSON 字符串；结构化输出（FileList/Table/...）→ JSON 数组/对象
- stdout 纪律：数据走 stdout，诊断走 stderr
- 退出码传播：错误 → 非零退出码（Agent 据此判断成功/失败）

### MS2：沙盒与权限控制（Plan 008 + 009）

AI 生成的命令在受限环境执行，可审计、可撤销。这是 bash/PowerShell **永远不会做**的差异化护城河。

**命令级拦截（Plan 008）**：
```
ash -c "rm file" --deny rm           # 命令名黑名单
ash -c "ls" --allow ls               # 白名单（默认拒绝）
ash -c "git status" --no-exec        # 禁所有外部命令（内置仍可用）
ash -c "curl http://x" --no-network  # 禁网络命令（http_*/curl/wget/ssh...）
ash -c "touch f" --dry-run           # 预演：打印意图，不实际执行
ash -c "echo hi" --audit log.jsonl   # 审计日志（JSON lines）
ash -c "rm -rf /"                    # 危险模式自动拦截（rm -rf /、mkfs、fork bomb...）
```

**文件级拦截（Plan 009）**：
```
ash -c "touch /etc/x" --sandbox /tmp   # 路径沙盒：文件操作限制在目录内
ash -c "cat /etc/passwd" --sandbox /tmp  # 读越界也被拒
ash -c "cd .." --sandbox /tmp/sub       # cd 不能跳出沙盒
ash -c "echo hi > out" --sandbox /tmp   # 重定向目标也受约束
ash -c "touch f" --read-only            # 路径级只读（拦 std::fs 写）
```

- 符号链接穿透检测（canonicalize 后查边界）
- 中央 `Shell::resolve_path` 统一了 10 个命令的散落路径解析
- `--sandbox` 受安全策略约束（system() 调 shell 也走 dispatch，策略自动生效）
- 零策略 = 全放行（向后兼容）

### MS3：脚本编程能力（Plan 010 + 011）

AutoLang 已具备 `fn`/`if`/`for`/`loop`/`break`/`continue`/`return`/类型化变量/`&&`/`||`/`?.` 错误传播。本版补齐三项硬缺口：

**try/catch + while（Plan 010）**：
```auto
try {
    var c = 10 / 0
} catch (e) {
    print("caught: " + e)   // 捕获运行时错误
}
var i = 0
while (i < 3) { print(i); i = i + 1 }
```

**Shell 桥接（Plan 011）**：脚本能调 shell、设环境变量、控制退出码。
```auto
var out = system("echo hello")   // 调 shell，返回 stdout
print(out)                        // → hello
export("DEPLOY_ENV", "staging")  // 设环境变量
exit(42)                          // 脚本退出码 = 42
```

- `system(cmd)` 走 shell dispatch，自动受 `--sandbox`/`--deny` 约束
- 端到端部署脚本示例见 `examples/deploy.ash`

## 🔧 技术架构

### MS2 的两层拦截
- **命令级**：`SecurityPolicy`（allow/deny/no-exec/no-network/read-only/dry-run/audit/危险模式）在 `execute_single_command` 派发前统一检查
- **文件级**：`Shell::resolve_path(arg, for_write)` 集中路径解析 + canonicalize + 沙盒边界检查，10 个命令迁移到它

### MS3 的跨仓库实现
- **try/catch**：VM 异常表机制（`PUSH_HANDLER`/`POP_HANDLER` opcode + per-task handler 栈），3 个执行循环统一接入 `intercept_error`
- **shell 桥接**：`ShellHost` trait + AutoVM host 插槽（auto-lang 仓库），ash 用 raw `*mut Shell` 指针 + `begin_run`/`end_run` 解决"native 同步执行拿不到 &mut Shell"的借用冲突（单线程 unsafe，已隔离）

## ⚠️ 已知限制

- **持久 session 的 fn+循环 bug**（预存，非本版引入）：ash 脚本里循环/try 嵌套在 `fn` 内会栈溢出。规避：循环和 try/catch 放脚本顶层（`examples/deploy.ash` 演示）。根因在 auto-lang 仓库的持久 session codegen，留后续修复。
- **外部进程不受沙盒约束**：`--sandbox` 只约束 ash 内置命令的文件操作；外部进程（git 等）的 fs 访问需配合 `--no-exec` 或容器隔离（OS 级限制，已文档声明）。
- **`--read-only` 路径级**已实现，但 legacy fs.rs builtin 的次级路径未完全迁移（registry 命令优先且已迁移，影响极小）。
