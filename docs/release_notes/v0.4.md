## 概述

v0.4 是 ash 从"基础 shell"向**数据感知 shell**迈进的关键版本。核心变化是引入了**结构化管道**——命令之间传递的是带类型的表格/记录数据，而非纯文本。这让 ash 能像 Nushell 一样对 JSON/CSV 等数据做筛选、排序、聚合，并以表格形式渲染。

本版本还大幅补齐了 **POSIX 命令合规性**，并引入了自适应补全菜单、真彩色语法高亮、F1/F2/F3 模式切换、作业控制、ASH 脚本支持等日常体验改进。

## ✨ 亮点功能

### 1. 结构化管道（Atom Pipeline）

命令间传递的数据从纯文本升级为**带类型标签的结构化数据**（Atom）。`ls` 输出的是"文件列表"、`show` 解析出的是"表格"、`grep` 输出的是"匹配列表"——类型标签让渲染层自动选择最合适的展示方式（表格/记录/纯文本），也让下游命令能按字段操作。

```
# ls 输出结构化文件列表，自动渲染成表格
ls

# 表格可以在管道里被结构化命令处理
ls | sort -w size         # 按大小排序
ls | grep ".rs"           # 过滤出含 .rs 的行
show data.csv | sort -w age -r   # CSV 按字段降序
```

> 底层用自定义 Batom 二进制格式在命令间高效传递结构化数据。

### 2. show / open — 文件解析与 GUI 打开

v0.4 重新定义了 `open` 和 `show` 的分工（对齐 auto-os 生态约定）：

| 命令 | 用途 | 示例 |
|---|---|---|
| `show <file>` | 按扩展名解析文件进管道 | `show data.csv`（→ 表格）|
| `open <file>` | 用系统默认程序 GUI 打开 | `open report.pdf`（→ PDF 阅读器）|

`show` 支持自动嗅探格式（`.json`/`.csv`），未知扩展名回退为纯文本。可用 `--as` 强制指定格式：

```
show data.csv                  # CSV → 表格
show config.json               # JSON → 表格/记录
cat data.csv | show --as csv   # 管道模式
show notes.txt                 # 未知扩展名 → 纯文本
open .                         # 用资源管理器打开当前目录
```

### 3. sort — 按字段排序

`sort` 不再只是行排序器，现在支持对结构化表格按字段排序：

```
show data.csv | sort -w age        # 按字段名升序
show data.csv | sort -w age -r     # 降序
show data.csv | sort -w .age       # Auto 风格 .field 访问
ls | sort -w size                  # ls 的 size 字段（数值排序）
sort -k 2 -t , -n file.csv         # POSIX 风格按列号排序
```

字段名访问支持 `field` 和 `.field` 两种写法（等价）。数值类型字段（如 ls 的 `size`）自动按数值排序，无需 `-n`。

### 4. POSIX 命令合规（Plan 006）

补齐了一批传统 Unix 命令缺失的 POSIX 必选项：

| 命令 | 新增 flag |
|---|---|
| **echo** ⚠️ | 默认输出加结尾换行（POSIX）；`-n` 抑制换行 |
| **date** | `-u`（UTC 短形式）；`+format` 操作数（`date +"%Y-%m-%d"`）|
| **cut** | `-c`（字符）/`-b`（字节）/`-s`（抑制无分隔符行）+ 开区间范围 |
| **grep** | `-E`（扩展正则）/`-F`（字面串）/`-q`（静默）/`-x`（整行）/`-w`（词匹配）|
| **touch** | `-a`（access time）/`-m`（mtime）/`-r`（参考文件）|
| **mv/rm/cp** | `-i`（交互式拒绝覆盖）；`-R`（大写递归）；`-v`（verbose 短形式）|
| **mkdir** | `-m`（设置权限模式，Unix）|
| **ls** | `-A`（隐藏文件但排除 `.`/`..`）|

### 5. 自适应补全菜单（AshMenu）

Tab 补全现在有智能的菜单：
- **自适应布局**：候选项少时用紧凑网格，有描述时自动切到描述列表
- **option 名补全**：`sort -<Tab>` 现在能列出 `-w`/`-k`/`-t`（之前只列 flag 不列 option）
- **flag 描述**：补全候选带描述（`-w  Sort records by FIELD name`），不再只是裸 flag
- **`-` 与 `--` 分离**：`-<Tab>` 只出短形式，`--<Tab>` 只出长形式

### 6. 真彩色语法高亮 + 主题

- 24-bit 真彩色检测，不支持时自动降采样到 256 色
- 可配置的语法高亮主题（`color` 命令）
- 命令/字符串/数字/操作符等不同着色

### 7. F1/F2/F3 模式切换 + AI 模式

| 按键 | 模式 | 说明 |
|---|---|---|
| F1 | Shell | 传统 shell 模式（默认）|
| F2 | Auto | AutoLang 表达式模式 |
| F3 | AI | AI 辅助模式（F3 AI）|

支持锁定模式（`Alt+数字` 强制切换）。

### 8. 作业控制 + 交互增强

- **作业控制**：`cmd &`（后台）、`jobs`、`fg`、`bg`、`Ctrl+Z` 挂起
- **Ctrl+C 保护**：防止误操作中断 shell
- **Ctrl+R**：历史命令搜索菜单
- **Ctrl+E**：在外部编辑器编辑当前输入

### 9. ASH 脚本与 shell 语法

- `>` shell 语法（`ls > files.txt` 重定向）
- 变量插值（`$name`、`${name}`）
- 特殊变量：`$?`（退出码）、`$@`（所有参数）、`$#`（参数数）、`$!`（后台 PID）
- 算术展开 `$((1+2))`、花括号展开 `{a,b,c}`、命令替换 `$(cmd)`、进程替换 `<(cmd)`
- `~user` 展开、env 前缀作用域（`FOO=bar cmd`）

### 10. 其他

- **统一配置系统**：`~/.config/ash.at`（AutoLang 格式）
- **env 持久化**：`env -save` / `env -load`，启动自动加载
- **声明式外部命令补全**：git/cargo/docker/npm/ssh 等，加 `completions` 管理命令
- **ash-gui 工作区**：AutoUI 版本的脚手架（Plan 330）

## ⚠️ 破坏性变更

### `echo` 默认现在输出结尾换行（POSIX 行为）

**之前**（v0.3）：`echo hello` → `hello`（无换行）
**现在**（v0.4）：`echo hello` → `hello\n`

这是对齐 POSIX 标准的必要变更。如果你的脚本依赖 echo 不加换行的旧行为，使用 `echo -n`。

### `open` 命令语义变更

v0.3 的 `open` 是"解析文件进管道"。v0.4 起：
- `open` = **用系统默认程序 GUI 打开**（对齐 auto-os 生态）
- `show` = **解析文件进管道**（原 `open` 的行为，改名）

迁移：`open data.csv | ...` → `show data.csv | ...`

## 🐛 关键修复

| 问题 | 修复 |
|---|---|
| `cd ~/子目录` / `ls ~` 在 Windows 失效 | 修复两处 `~` 展开根因（PathBuf::join 陷阱 + parse_args 吃反斜杠）|
| `parse_args` 吃掉 Windows 反斜杠路径 | 裸文本反斜杠改为字面字符（双引号内仍可转义）|
| prompt 只显示最后 3 层目录 | 改为显示完整路径（home 缩写为 `~`）|
| `sort -w size` 不排序（数值字段当缺失）| 数值类型字段按值排序 |
| 表头列名与字段名不一致（`Age` vs `age`）| 表头改为原样显示（不再大写化）|

- **ash-core**：纯逻辑层（补全、解析、管道类型），零终端依赖
