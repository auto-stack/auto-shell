## 路线 C：OS 管道流式 + less 流式消费

### 原理

```
show file.rs | less

主进程 (ash REPL):
  ① 检测到 show 是注册命令且非管线末尾
  ② spawn 子进程: ash -c "show file.rs"  → stdout 连 OS 管道
  ③ 返回 ExternalStream 给 less
  ④ less 从管道逐行读取 (后台线程) + 交互分页 (主线程)

子进程 (ash -c "show file.rs"):
  ① is_pipeline_last = true (单命令执行)
  ② 逐行高亮 → highlight_code_to_writer → 写入 stdout (OS 管道)
  ③ 与 less 的读取并发运行 ← 这是流式的关键!
```

### 改动文件

#### 1. `cmd.rs` — Command trait 新增 opt-in 方法

```rust
/// 声明此命令可安全地作为 `ash -c` 子进程运行以产生 OS 管道流。
/// 管线引擎在 !is_last 时优先 spawn 这类命令，而非物化为 Text。
fn is_streamable_producer(&self) -> bool { false }
```

默认 `false`（不影响现有命令）。`ShowCommand` override 为 `true`。

#### 2. `shell.rs` — `execute_pipeline_with_auto` 新增 Phase 0

在 Phase 1 (registry dispatch) 之前插入：

```
if !is_last && registered_cmd.is_streamable_producer() {
    // 用 current_exe() 获取 ash 二进制路径
    // spawn: <ash> -c "<cmd>"
    // 返回 ExternalStream → 跳到下一个命令
}
```

复用已有的 `external::spawn_external_stream()`，不新建管道逻辑。

#### 3. `less.rs` — `run_atom` 重写，支持流式输入

核心改动：`LessCommand` override `run_atom()`，检测 `AtomPipeline::ExternalStream`：

```rust
fn run_atom(&self, args, input, shell) -> Result<AtomPipeline> {
    if let AtomPipeline::ExternalStream(es) = input {
        // 后台线程: 逐行读取管道 → 追加到共享 Vec
        let lines = Arc::new(Mutex::new(Vec::new()));
        let done = Arc::new(AtomicBool::new(false));
        thread::spawn(move || {
            for line in es.lines() { lines.lock().push(line?); }
            done.store(true);
        });
        // 主线程: 启动分页器
        // - 立即显示已有行 (~30行就出首屏)
        // - 用户按键时检查是否有新行到达 → 增量更新
        // - 状态栏显示 "(loading... N lines)" 直到 done
        run_streaming_pager(lines, done)?;
        return Ok(AtomPipeline::Empty);
    }
    // 非 ExternalStream → 走原有 Text 路径
    let legacy = atom_to_pipeline_data(input);
    ...
}
```

新增 `StreamingPager` 结构（基于现有 `Pager`，但 lines 来自 `Arc<Mutex<Vec<String>>>`）：
- 每次渲染时 lock → 复制当前可见行 → unlock → 写终端
- 按键间隙轮询 `lines.lock().len()` 是否增长
- `G`（跳末尾）时若 `!done`，显示 "loading..." 并等待

#### 4. `show.rs` — 确认子进程上下文正确

`show` 作为 `ash -c "show file.rs"` 子进程运行时：
- `is_pipeline_last()` = `true`（单命令执行路径）
- `Format::Code` → `highlight_code_to_writer` 流式写 stdout ✅
- 无需改动，已兼容

### 效果

| 场景 | 改动前 | 改动后 |
|------|:------:|:------:|
| `show file.rs \| less` | 等 3s 全高亮 → less 显示 | **并发**: show 高亮 + less 读取，首屏 ~0.05s |
| `show -p file.rs` | 0.03s (CodePager) | 不变 |
| `cat big.log \| less` | 外部命令，已有流式 | 不变 |
| `show file.rs` (单独) | 流式 stdout | 不变 |
